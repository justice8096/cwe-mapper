# CWE Top 25 Most Dangerous Software Weaknesses (2024)

Detailed analysis of the 25 most dangerous CWEs with detection patterns, vulnerable code examples, and remediation guidance.

## 1. CWE-787: Out-of-Bounds Write

**Severity**: CRITICAL | **CVSS**: 9.8

**Description**: Writing data beyond the boundaries of allocated memory. Common in C/C++ but possible in any language without bounds checking.

**Detection Pattern**:
```regex
(memcpy|strcpy|strcat|memmove)\s*\(\s*[^,]+\s*,\s*[^,]+\s*,\s*[^)]*\)
Array access: \[[^\]]*\]\s*=
```

**Vulnerable Code** (C):
```c
char buffer[256];
strcpy(buffer, user_input);  // No bounds check!
```

**Vulnerable Code** (JavaScript):
```javascript
const arr = new Array(10);
for (let i = 0; i <= arr.length; i++) {  // Off-by-one
  arr[i] = data[i];
}
```

**Remediation Pattern**:
```c
// Use safe functions
char buffer[256];
strncpy(buffer, user_input, sizeof(buffer) - 1);
buffer[sizeof(buffer) - 1] = '\0';
```

**Remediation** (JavaScript):
```javascript
const arr = new Array(10);
for (let i = 0; i < arr.length; i++) {  // Correct bounds
  arr[i] = data[i];
}
```

**Framework Mapping**:
- OWASP A02: Cryptographic Failures
- NIST: SI-10 (Information System Monitoring), SI-4 (Information System Monitoring)
- MITRE ATT&CK: T1055 (Process Injection), T1190 (Exploit Public-Facing Application)

## 2. CWE-79: Cross-Site Scripting (XSS)

**Severity**: HIGH | **CVSS**: 6.1-7.3

**Description**: Unvalidated user input reflected in HTML output, allowing script injection in victim's browser.

**Detection Pattern**:
```regex
innerHTML\s*=
dangerouslySetInnerHTML
eval\s*\(
Function\s*\(.*user.*\)
template.*\$\{.*user.*\}
```

**Vulnerable Code** (JavaScript):
```javascript
const name = req.query.name;
res.send(`<h1>Hello ${name}</h1>`);  // Direct interpolation
```

**Vulnerable Code** (Python Flask):
```python
@app.route('/greet')
def greet():
    name = request.args.get('name')
    return f'<h1>Hello {name}</h1>'  # Jinja auto-escape disabled?
```

**Remediation Pattern**:
```javascript
// Escape HTML entities
const name = req.query.name;
const escaped = name.replace(/&/g, '&amp;')
                     .replace(/</g, '&lt;')
                     .replace(/>/g, '&gt;');
res.send(`<h1>Hello ${escaped}</h1>`);
```

**Remediation** (Python):
```python
from markupsafe import escape
@app.route('/greet')
def greet():
    name = request.args.get('name')
    return f'<h1>Hello {escape(name)}</h1>'
```

**Framework Mapping**:
- OWASP A03: Injection
- OWASP LLM02: Unsafe Plugin Execution
- NIST: SI-10
- EU AI Act: Article 15 (Risk Assessment)

## 3. CWE-89: SQL Injection

**Severity**: CRITICAL | **CVSS**: 9.8

**Description**: Unvalidated user input concatenated into SQL queries, allowing attacker to modify query logic.

**Detection Pattern**:
```regex
"SELECT.*"\s*\+\s*[a-zA-Z_]
'SELECT.*'\s*\+\s*[a-zA-Z_]
query\s*=\s*f"SELECT.*\{
string\.format.*SELECT.*
```

**Vulnerable Code** (Java):
```java
String query = "SELECT * FROM users WHERE id = " + userId;
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery(query);
```

**Vulnerable Code** (Python):
```python
query = f"SELECT * FROM users WHERE email = '{email}'"
cursor.execute(query)
```

**Remediation Pattern**:
```java
String query = "SELECT * FROM users WHERE id = ?";
PreparedStatement pstmt = conn.prepareStatement(query);
pstmt.setInt(1, userId);
ResultSet rs = pstmt.executeQuery();
```

**Remediation** (Python):
```python
query = "SELECT * FROM users WHERE email = %s"
cursor.execute(query, (email,))
```

**Framework Mapping**:
- OWASP A03: Injection
- NIST: SI-10, CM-6 (Configuration Management)
- EU AI Act: Article 15
- ISO 27001: A8.1 (Cryptography)
- SOC 2: CC7.1 (System Monitoring)

## 4. CWE-416: Use After Free

**Severity**: CRITICAL | **CVSS**: 8.8

**Description**: Using memory after it has been deallocated. Can lead to arbitrary code execution.

**Detection Pattern**:
```regex
free\s*\(\s*ptr\s*\).*ptr->
delete\s+ptr\s*;.*ptr->
dealloc.*\*ptr\s*=\s*\[
drop\s*\(.*\).*use of .* after move
```

**Vulnerable Code** (C):
```c
int *ptr = malloc(sizeof(int));
*ptr = 42;
free(ptr);
printf("%d", *ptr);  // Use after free!
```

**Vulnerable Code** (Rust):
```rust
let s = String::from("hello");
drop(s);
println!("{}", s);  // Compiler error (good!)
```

**Remediation** (C):
```c
int *ptr = malloc(sizeof(int));
if (ptr) {
  *ptr = 42;
  printf("%d", *ptr);
  free(ptr);
  ptr = NULL;  // Null out after free
}
```

**Remediation** (Rust):
```rust
let s = String::from("hello");
println!("{}", s);
drop(s);  // Only use after drop ends scope
```

**Framework Mapping**:
- OWASP A02: Cryptographic Failures
- NIST: SI-4 (Information System Monitoring)
- MITRE ATT&CK: T1055

## 5. CWE-78: OS Command Injection

**Severity**: CRITICAL | **CVSS**: 9.8

**Description**: Unvalidated user input passed to OS command execution functions.

**Detection Pattern**:
```regex
os\.system\s*\(\s*[a-zA-Z_]+
exec\s*\(\s*[a-zA-Z_]+
`.*\$\{.*user.*\}`
subprocess\.call\s*\([^,]+\s*\+
```

**Vulnerable Code** (Python):
```python
filename = request.args.get('file')
os.system(f'cat {filename}')  # Command injection!
```

**Vulnerable Code** (JavaScript):
```javascript
const cmd = `ping -c 1 ${userHost}`;
exec(cmd, (error, stdout) => {
  res.send(stdout);
});
```

**Remediation Pattern** (Python):
```python
from shlex import quote
filename = request.args.get('file')
os.system(f'cat {quote(filename)}')  # Safer but still risky
```

**Remediation** (JavaScript):
```javascript
const { execFile } = require('child_process');
execFile('ping', ['-c', '1', userHost], (error, stdout) => {
  res.send(stdout);
});
```

**Framework Mapping**:
- OWASP A03: Injection
- NIST: SI-10, SI-4
- EU AI Act: Article 15

## 6. CWE-20: Improper Input Validation

**Severity**: HIGH | **CVSS**: 7.5

**Description**: Failure to validate, filter, or sanitize user-supplied input before processing.

**Detection Pattern**:
```regex
request\.(args|form|json)\[.*\]
req\.(params|query|body)\.[a-zA-Z_]+
argv\[
sys\.argv\[
```

**Vulnerable Code** (Python):
```python
age = int(request.args.get('age'))
if age < 150:  # No upper bound check!
    process_user(age)
```

**Vulnerable Code** (JavaScript):
```javascript
const id = req.query.id;
const user = db.getUser(id);  // ID could be SQL injection
```

**Remediation Pattern**:
```python
from marshmallow import Schema, fields, ValidationError
class AgeSchema(Schema):
    age = fields.Int(required=True, validate=lambda x: 0 <= x <= 150)

try:
    data = AgeSchema().load(request.args)
    process_user(data['age'])
except ValidationError as err:
    return {'errors': err.messages}
```

**Framework Mapping**:
- OWASP A03: Injection
- NIST: CM-6, SI-10

## 7. CWE-125: Out-of-Bounds Read

**Severity**: HIGH | **CVSS**: 7.5

**Description**: Reading data beyond buffer boundaries, causing information disclosure.

**Detection Pattern**:
```regex
\[[^\]]*\]\s*without bounds check
for\s*\(\s*[^;]+<=[^;]+\.length
memcpy.*user.*size
```

**Vulnerable Code** (C):
```c
char buffer[256];
memcpy(buffer, untrusted, untrusted_len);  // No bounds!
```

**Vulnerable Code** (JavaScript):
```javascript
const arr = [1, 2, 3];
console.log(arr[userIndex]);  // What if index > 2?
```

**Remediation**:
```c
char buffer[256];
size_t max_size = sizeof(buffer) - 1;
memcpy(buffer, untrusted, min(untrusted_len, max_size));
```

**Framework Mapping**:
- OWASP A01: Broken Access Control
- NIST: SI-4

## 8. CWE-22: Path Traversal

**Severity**: HIGH | **CVSS**: 7.5

**Description**: Unvalidated path input allows access to files outside intended directory.

**Detection Pattern**:
```regex
\.\./
\.\.\\\
request\.(args|form|json)\['(file|path|dir|filename)'\]
open\s*\(\s*[a-zA-Z_]+
Path\s*\(\s*[a-zA-Z_]+
```

**Vulnerable Code** (Python):
```python
filename = request.args.get('file')
with open(f'/uploads/{filename}') as f:  # Path traversal!
    return f.read()
```

**Vulnerable Code** (Node.js):
```javascript
const path = req.query.path;
const file = `/uploads/${path}`;  // Allows ../../../etc/passwd
sendFile(file);
```

**Remediation Pattern**:
```python
from pathlib import Path
filename = request.args.get('file')
base_path = Path('/uploads').resolve()
file_path = (base_path / filename).resolve()
if not str(file_path).startswith(str(base_path)):
    abort(403)  # Path escape attempt
with open(file_path) as f:
    return f.read()
```

**Framework Mapping**:
- OWASP A01: Broken Access Control
- NIST: AC-3 (Access Control)

## 9. CWE-352: Cross-Site Request Forgery (CSRF)

**Severity**: MEDIUM | **CVSS**: 6.5

**Description**: State-changing request made on behalf of victim without their knowledge/consent.

**Detection Pattern**:
```regex
POST|PUT|DELETE.*without.*token|csrf
form\s+method=["']post["']\s+(?!.*csrf)
fetch\s*\(\s*[^,]+\s*,\s*\{\s*method\s*:\s*["'](POST|PUT|DELETE)
```

**Vulnerable Code** (Flask):
```python
@app.route('/transfer', methods=['POST'])
def transfer():
    amount = request.form.get('amount')
    recipient = request.form.get('recipient')
    process_transfer(amount, recipient)  # No CSRF token!
```

**Vulnerable Code** (HTML):
```html
<form method="POST" action="/deleteAccount">
  <input type="hidden" name="confirm" value="yes">
  <button>Delete My Account</button>
</form>
```

**Remediation Pattern**:
```python
from flask_wtf.csrf import csrf_protect
@app.route('/transfer', methods=['POST'])
@csrf_protect
def transfer():
    amount = request.form.get('amount')
    recipient = request.form.get('recipient')
    process_transfer(amount, recipient)
```

**Framework Mapping**:
- OWASP A01: Broken Access Control
- NIST: SI-10, SI-4

## 10. CWE-434: Unrestricted Upload of File with Dangerous Type

**Severity**: HIGH | **CVSS**: 8.8

**Description**: Uploading files without type validation or execution restrictions.

**Detection Pattern**:
```regex
request\.files\[
\.filename
\.save\(\s*\)
upload.*\.exe|\.php|\.jsp
```

**Vulnerable Code** (Flask):
```python
@app.route('/upload', methods=['POST'])
def upload():
    file = request.files['file']
    file.save(f'/uploads/{file.filename}')  # No validation!
    return 'File uploaded'
```

**Vulnerable Code** (Node.js):
```javascript
app.post('/upload', (req, res) => {
  const file = req.files.upload;
  file.mv(`/uploads/${file.name}`);  // PHP executable?
});
```

**Remediation Pattern**:
```python
import os
from werkzeug.utils import secure_filename
ALLOWED_EXTENSIONS = {'txt', 'pdf', 'png', 'jpg'}

@app.route('/upload', methods=['POST'])
def upload():
    file = request.files['file']
    if '.' in file.filename:
        ext = file.filename.rsplit('.', 1)[1].lower()
        if ext not in ALLOWED_EXTENSIONS:
            return 'File type not allowed', 400

    filename = secure_filename(file.filename)
    file.save(os.path.join('/uploads', filename))
```

**Framework Mapping**:
- OWASP A04: Insecure Design
- NIST: SI-10, SI-7 (Software, Firmware, and Information Integrity)

## 11. CWE-862: Missing Authorization

**Severity**: CRITICAL | **CVSS**: 9.1

**Description**: No check to ensure user has permission to perform requested action.

**Detection Pattern**:
```regex
def\s+(edit|delete|update|admin).*:\s*(?!.*authz|.*permission|.*role)
if\s+is_authenticated.*(?!.*is_admin|.*permission)
@(app|api)\.route.*(?!.*@require_role|.*@require_permission)
```

**Vulnerable Code** (Flask):
```python
@app.route('/user/<user_id>/delete', methods=['POST'])
@login_required
def delete_user(user_id):
    user = User.query.get(user_id)
    db.session.delete(user)  # No authz check!
    db.session.commit()
```

**Vulnerable Code** (Express):
```javascript
app.delete('/admin/users/:id', isLoggedIn, (req, res) => {
  User.findByIdAndRemove(req.params.id);  // Only auth, no authz
  res.send('Deleted');
});
```

**Remediation Pattern**:
```python
@app.route('/user/<user_id>/delete', methods=['POST'])
@login_required
def delete_user(user_id):
    user = User.query.get(user_id)
    if user.id != current_user.id and not current_user.is_admin:
        abort(403)  # Authorization check
    db.session.delete(user)
    db.session.commit()
```

**Framework Mapping**:
- OWASP A01: Broken Access Control
- NIST: AC-2, AC-6 (Least Privilege)
- EU AI Act: Article 35 (Documentation)
- SOC 2: CC6.1 (Access Controls)

## 12. CWE-476: NULL Pointer Dereference

**Severity**: MEDIUM | **CVSS**: 5.5

**Description**: Using a null/nil pointer without checking if it's null, causing crash.

**Detection Pattern**:
```regex
if\s+\([^)]*ptr[^)]*\)\s+\{[^}]*ptr->
try\s+\{[^}]*obj\.[a-zA-Z_]+\}
\?\.?\s+[a-zA-Z_]+\.[a-zA-Z_]+(?!=)
```

**Vulnerable Code** (Java):
```java
User user = userRepository.findById(userId);
user.setActive(true);  // user could be null!
```

**Vulnerable Code** (Python):
```python
result = db.query(User).filter_by(id=user_id).first()
result.update({'active': True})  # result could be None
```

**Remediation Pattern**:
```java
User user = userRepository.findById(userId);
if (user != null) {
    user.setActive(true);
    userRepository.save(user);
}
```

**Framework Mapping**:
- OWASP A02: Cryptographic Failures
- NIST: SI-4

## 13. CWE-287: Improper Authentication

**Severity**: CRITICAL | **CVSS**: 9.8

**Description**: Weak or missing authentication mechanisms allowing unauthorized access.

**Detection Pattern**:
```regex
password\s*==\s*user_password
api_key\s*in\s*request\.headers
hardcoded.*password|api.?key
secret\s*=\s*["']
```

**Vulnerable Code** (Python):
```python
@app.route('/login', methods=['POST'])
def login():
    username = request.form.get('username')
    password = request.form.get('password')
    if username == 'admin' and password == 'admin123':  # Hardcoded!
        session['user'] = username
```

**Vulnerable Code** (Node.js):
```javascript
app.post('/auth', (req, res) => {
  const apiKey = req.headers['x-api-key'];
  if (apiKey === 'sk-12345') {  // Hardcoded key!
    res.send({authenticated: true});
  }
});
```

**Remediation Pattern**:
```python
from werkzeug.security import generate_password_hash, check_password_hash
import os

@app.route('/login', methods=['POST'])
def login():
    username = request.form.get('username')
    password = request.form.get('password')
    user = User.query.filter_by(username=username).first()
    if user and check_password_hash(user.password_hash, password):
        session['user'] = username
```

**Framework Mapping**:
- OWASP A07: Authentication Failures
- NIST: IA-2 (Authentication), IA-5 (Authentication Mechanisms)
- ISO 27001: A9.2 (User Access Management)

## 14. CWE-190: Integer Overflow

**Severity**: MEDIUM | **CVSS**: 6.5

**Description**: Integer arithmetic results exceed representation limits.

**Detection Pattern**:
```regex
size\s*\+\s*
len\s*\*\s*
width\s*\*\s*height
\*\s*sizeof
```

**Vulnerable Code** (C):
```c
uint32_t size = user_size * sizeof(struct Item);
void *buffer = malloc(size);  // If user_size is large, size overflows
```

**Vulnerable Code** (Java):
```java
int total = quantity * price;  // Could overflow
Account.debit(total);
```

**Remediation Pattern**:
```c
#include <stdint.h>
#include <limits.h>

if (user_size > (UINT32_MAX / sizeof(struct Item))) {
    return -1;  // Overflow would occur
}
uint32_t size = user_size * sizeof(struct Item);
void *buffer = malloc(size);
```

**Framework Mapping**:
- OWASP A02: Cryptographic Failures
- NIST: SI-4

## 15. CWE-502: Deserialization of Untrusted Data

**Severity**: HIGH | **CVSS**: 8.1

**Description**: Deserialization of untrusted data can execute arbitrary code via gadget chains.

**Detection Pattern**:
```regex
pickle\.loads
ObjectInputStream\.readObject
JSON\.parse.*eval
fromJSON\s*\(
XMLDecoder
```

**Vulnerable Code** (Python):
```python
import pickle
user_data = request.get_json()
obj = pickle.loads(user_data['object'])  # RCE via gadget chain!
```

**Vulnerable Code** (Java):
```java
ObjectInputStream ois = new ObjectInputStream(userInput);
Object obj = ois.readObject();  // Dangerous with Commons Collections
```

**Remediation Pattern** (Python):
```python
import json
user_data = request.get_json()
obj = json.loads(user_data['object'])  # Safe: JSON only supports primitives
```

**Remediation** (Java):
```java
import com.fasterxml.jackson.databind.ObjectMapper;
ObjectMapper mapper = new ObjectMapper();
mapper.activateDefaultTyping(mapper.getPolymorphicTypeValidator());
Object obj = mapper.readValue(userInput, Object.class);
```

**Framework Mapping**:
- OWASP A08: Data Integrity Failures
- NIST: SI-10, SI-7
- EU AI Act: Article 15
- SOC 2: CC6.1
